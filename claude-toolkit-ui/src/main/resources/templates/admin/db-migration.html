DB 마이그레이션 가이드

기본 H2 파일 DB에서 PostgreSQL / MySQL / Oracle 11g 로 이전하는 절차입니다. 본 가이드는 방법만 안내하며 실제 마이그레이션은 관리자가 직접 수행해야 합니다.

현재 연결된 데이터베이스
로딩 중...
1PostgreSQL 서버 준비 (Docker 예시)
`docker-compose.yml`에 이미 정의된 `postgresql` 프로파일을 사용하면 가장 간단합니다.
docker-compose --profile postgresql up -d db-pg
또는 직접 PostgreSQL 14+ 인스턴스 실행:
docker run -d --name claude-pg \
  -e POSTGRES_DB=claude_toolkit \
  -e POSTGRES_USER=claude \
  -e POSTGRES_PASSWORD=claude1234 \
  -p 5432:5432 \
  postgres:14-alpine
2환경변수 설정 후 재기동
Docker 컴포즈 환경:
DB_TYPE=postgresql \
DB_HOST=db-pg \
DB_PORT=5432 \
DB_NAME=claude_toolkit \
DB_USERNAME=claude \
DB_PASSWORD=claude1234 \
docker-compose up -d claude-toolkit
앱은 자동으로 application-postgresql.yml 프로파일을 로드합니다.
3스키마 자동 생성 확인
JPA ddl-auto: update로 모든 테이블이 자동 생성됩니다. 최초 기동 시 admin 계정이 자동 생성됩니다.
application-postgresql.yml 설정 (참고)
spring:
  datasource:
    url: jdbc:postgresql://${DB_HOST:localhost}:${DB_PORT:5432}/${DB_NAME:claude_toolkit}
    driver-class-name: org.postgresql.Driver
    username: ${DB_USERNAME:claude}
    password: ${DB_PASSWORD:claude1234}
    hikari:
      maximum-pool-size: 10
      minimum-idle: 2
  jpa:
    hibernate:
      ddl-auto: update
    properties:
      hibernate:
        dialect: org.hibernate.dialect.PostgreSQL10Dialect
1MySQL 서버 준비
MySQL 8.0+ 권장. 프로젝트에 포함된 `mysql` 프로파일 활용:
docker-compose --profile mysql up -d db
2데이터베이스 생성 및 권한
CREATE DATABASE claude_toolkit CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'claude'@'%' IDENTIFIED BY 'claude1234';
GRANT ALL PRIVILEGES ON claude_toolkit.* TO 'claude'@'%';
FLUSH PRIVILEGES;
3환경변수로 재기동
DB_TYPE=mysql \
DB_HOST=db \
DB_PORT=3306 \
DB_NAME=claude_toolkit \
DB_USERNAME=claude \
DB_PASSWORD=claude1234 \
docker-compose up -d claude-toolkit
⚠️ 주의: MySQL 8.0+ 필수. 5.7 이하는 JSON 타입 및 일부 기능 미지원. 문자셋은 반드시 utf8mb4를 사용하여 한글 4byte 문자(이모지 포함)를 지원하세요.
application-mysql.yml 설정 (참고)
spring:
  datasource:
    url: jdbc:mysql://${DB_HOST:localhost}:${DB_PORT:3306}/${DB_NAME:claude_toolkit}?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Seoul&characterEncoding=UTF-8
    driver-class-name: com.mysql.cj.jdbc.Driver
    username: ${DB_USERNAME:claude}
    password: ${DB_PASSWORD:claude1234}
  jpa:
    hibernate:
      ddl-auto: update
    properties:
      hibernate:
        dialect: org.hibernate.dialect.MySQL8Dialect
⚠️ Oracle 11g 주요 제약사항:
  • BOOLEAN 타입 미지원 → NUMBER(1) 로 자동 매핑
  • TEXT/VARCHAR 4000+ → CLOB 필수
  • AUTO_INCREMENT 없음 → 시퀀스(SEQUENCE) 기반 ID 생성
  • 예약어 주의 (예: level, comment, user)
  • Hibernate Dialect는 Oracle10gDialect 사용 (11g 호환)
1Oracle 11g 인스턴스 준비
Oracle XE 11g 또는 Enterprise Edition. 스키마 사용자 생성:
-- SYSDBA로 접속
CREATE USER CLAUDE_TK IDENTIFIED BY "Claude1234!";
GRANT CONNECT, RESOURCE TO CLAUDE_TK;
GRANT UNLIMITED TABLESPACE TO CLAUDE_TK;
GRANT CREATE SEQUENCE TO CLAUDE_TK;
GRANT CREATE TABLE TO CLAUDE_TK;
2ojdbc 드라이버 추가
프로젝트 pom.xml에 Oracle JDBC 의존성 추가 필요:
<dependency>
    <groupId>com.oracle.database.jdbc</groupId>
    <artifactId>ojdbc8</artifactId>
    <version>21.5.0.0</version>
    <scope>runtime</scope>
</dependency>
3application-oracle.yml 생성
신규 프로파일 파일을 src/main/resources/application-oracle.yml로 생성:
spring:
  datasource:
    url: jdbc:oracle:thin:@//${DB_HOST:localhost}:${DB_PORT:1521}/${DB_NAME:XE}
    driver-class-name: oracle.jdbc.OracleDriver
    username: ${DB_USERNAME:CLAUDE_TK}
    password: ${DB_PASSWORD:Claude1234!}
    hikari:
      maximum-pool-size: 10
      minimum-idle: 2
      connection-timeout: 30000
  jpa:
    hibernate:
      ddl-auto: update
    properties:
      hibernate:
        dialect: org.hibernate.dialect.Oracle10gDialect
        jdbc:
          lob:
            non_contextual_creation: true
        default_batch_fetch_size: 30
4환경변수 설정 후 재기동
SPRING_PROFILES_ACTIVE=oracle \
DB_HOST=oracle-db \
DB_PORT=1521 \
DB_NAME=XE \
DB_USERNAME=CLAUDE_TK \
DB_PASSWORD=Claude1234! \
java -jar claude-toolkit-ui.jar
🔧 호환성 점검 포인트:
  • JPA가 BOOLEAN을 NUMBER(1)로 매핑하는지 확인
  • TEXT 컬럼(CLOB)이 제대로 읽고 쓰기 가능한지 테스트
  • ddl-auto: update는 첫 기동 시 테이블 자동 생성
  • 이미 H2에 데이터가 있다면 별도 이관 작업 필요 (백업/복원 탭 참고)
1H2 데이터 백업
ADMIN 메뉴에서 전체 백업 ZIP 다운로드 가능:
/admin/backup (메뉴 → 백업 / 복원)
또는 H2 파일 직접 복사:
# Docker 컨테이너 외부로 복사
docker cp claude-toolkit:/root/.claude-toolkit/ ./backup-h2/
2H2 데이터를 SQL로 덤프 (H2 Console 활용)
-- H2 Console (http://localhost:8082) 또는 SQL 도구에서
SCRIPT TO '/tmp/h2-dump.sql';
생성된 SQL 파일을 목표 DB 방언에 맞게 편집 후 import합니다. H2와 대상 DB 문법 차이(ex. AUTO_INCREMENT vs SEQUENCE, TEXT vs CLOB)에 주의.
3대체 방법: 분석 이력만 CSV로 내보내기
전체 데이터가 아닌 분석 이력만 필요한 경우:
/history → 상단 "CSV 내보내기" 버튼
💡 권장 절차:
  1. H2 백업 ZIP 다운로드 (안전 백업)
  2. 새 DB에 프로파일 활성화 후 앱 재기동 (스키마 자동 생성)
  3. 신규 DB로 시작 → 기존 이력이 꼭 필요하면 수동 INSERT 또는 H2 SCRIPT 활용
  4. 운영 환경에서는 작업 전 기존 시스템을 점검 모드로 전환 권장
